# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

tsai is a state-of-the-art deep learning library for time series and sequential data, built on top of PyTorch and fastai. It supports classification, regression, forecasting, and imputation tasks.

## Claude Code Workflow (IMPORTANT)

**Do NOT directly edit any `.ipynb` or `.py` files in this repository.** Instead:

1. **Propose changes in the terminal** - Output the code changes as text that the user can review
2. **User will copy and paste** - The user will manually apply the changes to the appropriate files
3. **This applies to all code files** - Including notebooks in `nbs/` and any other Python files in `tsai/`

This workflow ensures the user maintains full control over file modifications and can review all changes before applying them.

## Development Setup

```bash
# Clone and install with dev dependencies
git clone https://github.com/timeseriesAI/tsai
pip install -e "tsai[dev]"

# Install nbdev git hooks (required for notebook development)
nbdev_install_hooks
```

## Build and Test Commands

```bash
# Run all tests, export notebooks to Python, and clean notebooks
nbdev_prepare

# Run tests only
nbdev_test

# Export notebooks to Python modules
nbdev_export

# Clean notebooks (remove outputs and metadata)
nbdev_clean

# Build documentation
nbdev_docs
```

## Architecture

### nbdev-Based Development

This project uses **nbdev** for literate programming. All source code lives in Jupyter notebooks under `nbs/` and is exported to Python modules in `tsai/`.
**Never manually edit files in the `tsai/` directory** - they are autogenerated from notebooks in `nbs/`. When changes are needed, Claude should propose the changes in the terminal for the user to apply manually to the corresponding `.ipynb` notebooks.

- Notebooks in `nbs/` are the source of truth
- Python files in `tsai/` are auto-generated (marked with `# AUTOGENERATED! DO NOT EDIT!`)
- Use `#|export` at the top of notebook cells to export code to modules
- Notebook naming: `NNN_module.submodule.ipynb` maps to `tsai/module/submodule.py`

### Key Directories

- `nbs/` - Source notebooks (edit these)
- `tsai/` - Auto-generated Python modules
- `tsai/models/` - Deep learning model implementations
- `tsai/data/` - Data loading, preprocessing, and transforms
- `tsai/callback/` - Training callbacks
- `tutorial_nbs/` - Tutorial notebooks (not part of the library)

### Import Patterns

```python
from tsai.all import *      # Import everything
from tsai.basics import *   # Import core functionality (smaller import)
```

### Core Classes

- **TSClassifier**, **TSRegressor**, **TSForecaster** (`tsai/tslearner.py`) - High-level learner classes that wrap fastai's Learner
- **TSTensor**, **TSDatasets**, **TSDataLoaders** (`tsai/data/core.py`) - Data containers
- **TSStandardize**, **SlidingWindow** (`tsai/data/preprocessing.py`, `tsai/data/preparation.py`) - Data transforms

### Input Data Format

Time series data uses 3D arrays: `[samples, variables, sequence_length]`

### Model Architecture Pattern

Models ending in `Plus` (e.g., `InceptionTimePlus`, `TSTPlus`) support flexible output heads for different tasks. Base models (e.g., `InceptionTime`, `TST`) are simpler implementations.

## Code Style

This project follows the **fastai code style**, not PEP8. Key differences:

- Short variable names are acceptable for numeric/interactive programming
- Compact code layout preferred over verbose
- See <https://docs.fast.ai/dev/style.html>

## Test Flags

Tests can use these flags (defined in `settings.ini`):

- `#|local` - Skip in CI, run locally only
- `#|slow` - Skip slow tests by default
- `#|extras` - Requires extra dependencies
- `#|onnx` - ONNX-related tests

## Configuration

Project settings are in `settings.ini`. Key settings:

- `lib_name = tsai`
- `min_python = 3.9`
- `requirements` - Hard dependencies
- `extra_requirements` - Optional dependencies (sktime, tsfresh, etc.)
